Principios WCAG

  • Perceptible
  • Operable
  • Comprensible
  • Robusto

Perceptible

Este principio enfatiza que la información y los componentes de la interfaz de usuario deben presentarse de manera que los usuarios puedan percibirlos, independientemente de sus capacidades sensoriales. Esto incluye proporcionar alternativas textuales para contenido no textual, ofrecer opciones para ajustar la presentación del contenido, y asegurar que el contenido audiovisual también sea accesible.

Operable

Se asegura de que los componentes de la interfaz y la navegación sean operables para todos los usuarios. Esto incluye hacer que todas las funcionalidades estén disponibles desde el teclado, dar suficiente tiempo para leer y usar el contenido, evitar el diseño de contenido que pueda provocar convulsiones, y proporcionar maneras de ayudar a los usuarios a navegar, encontrar contenido, y determinar dónde se encuentran.

Comprensible

Este principio establece que la información y el manejo de la interfaz de usuario deben ser comprensibles. Para ello, es importante que el contenido sea legible y comprensible, que las páginas web operen de manera predecible, y que los usuarios reciban asistencia para evitar y corregir errores.

Robusto

Busca que el contenido sea lo suficientemente robusto como para ser interpretado de manera fiable por una amplia variedad de usuarios, incluyendo aquellos que utilizan tecnologías asistivas. Esto implica asegurar la compatibilidad con las tecnologías asistivas actuales y futuras mediante el uso de estándares web consolidados.
